# Step 5: Self-Validation

Verify compliance with guidelines before finalizing.

## Quick Checklist

- [ ] `name`: ≤64 chars, lowercase-hyphen format, no leading/trailing hyphens
- [ ] `description`: ≤1024 chars, clear trigger conditions
- [ ] SKILL.md body: ≤500 lines
- [ ] Capability-only content (no behavior/rules)
- [ ] No forbidden keywords: "Always/Never/Should/Must"
- [ ] No extraneous files: README, CHANGELOG, INSTALLATION_GUIDE, etc.
- [ ] All reference files linked from SKILL.md with usage context

See [validation-checklist.md](./validation-checklist.md) for detailed rules and anti-patterns.
